Device¶

acDeviceStartStream¶

AC_ERROR AC_API acDeviceStartStream(acDevice hDevice)

acDeviceStartStream causes the device to begin streaming image/chunk data (acBuffer). It must be called before image or chunk data buffers are retrieved (acDeviceGetBuffer). The stream must be stopped (acDeviceStopStream) when no longer needed.

Return
  • Type: AC_ERROR
  • Error code for the function
  • Returns AC_ERR_SUCCESS (0) on success
Parameters
  • hDevice:
    • Type: acDevice
    • [In] parameter
    • A device

See
  • acBuffer
  • acDeviceGetBuffer
  • acDeviceStopStream

acDeviceStartStreamNumBuffersAndFlags¶

AC_ERROR AC_API acDeviceStartStreamNumBuffersAndFlags(acDevice hDevice, size_t numBuffers)

acDeviceStartStreamNumBuffersAndFlags causes the device to begin streaming image/chunk data (acBuffer) with given numBuffers and a streaming standard. It must be called before image or chunk data buffers are retrieved (acDeviceGetBuffer). The stream must be stopped (acDeviceStopStream) when no longer needed.

Return
  • Type: AC_ERROR
  • Error code for the function
  • Returns AC_ERR_SUCCESS (0) on success
Parameters
  • hDevice:
    • Type: acDevice
    • [In] parameter
    • A device
  • numBuffers:
    • Type: size_t
    • [In] parameter
    • Number of internal buffers to use in the acquisition engine

See
  • acBuffer
  • acDeviceGetBuffer
  • acDeviceStopStream

acDeviceStopStream¶

AC_ERROR AC_API acDeviceStopStream(acDevice hDevice)

acDeviceStopStream stops the device from streaming image/chunk data (acBuffer) and cleans up the stream. The stream must be stopped when streaming is no longer needed.

  • Type: AC_ERROR
  • Error code for the function

Parameters

    • Type: acDevice
    • [In] parameter

  • acBuffer

acDeviceGetBuffer¶

AC_ERROR AC_API acDeviceGetBuffer(acDevice hDevice, uint64_t timeout, acBuffer * phBuffer)

acDeviceGetBuffer retrieves a buffer (acBuffer) from the device. It must be called after the stream has started (acDeviceStartStream) and before the stream has stopped (acDeviceStopStream). Retrieved buffers must be requeued (acDeviceRequeueBuffer).

Return
  • Type: AC_ERROR
  • Error code for the function
  • Returns AC_ERR_SUCCESS (0) on success
Parameters
  • hDevice:
    • Type: acDevice
    • [In] parameter
    • A device
  • timeout:
    • Type: uint64_t
    • [In] parameter
    • Maximum time to wait for a buffer
  • phBuffer:
    • Type: acBuffer*
    • [Out] parameter
    • Next buffer in the output queue

See
  • acBuffer
  • acDeviceStartStream
  • acDeviceStopStream
  • acDeviceRequeueBuffer

acDeviceInitializeEvents¶

AC_ERROR AC_API acDeviceInitializeEvents(acDevice hDevice)

acDeviceInitializeEvents causes the underlying events engine to start listening for events. It must be called before waiting on events (acDeviceWaitOnEvent). The event infrastructure must be turned off (acDeviceDeinitializeEvents) when no longer needed.

Return
  • Type: AC_ERROR
  • Error code for the function
  • Returns AC_ERR_SUCCESS (0) on success
Parameters
  • hDevice:
    • Type: acDevice
    • [In] parameter
    • A device

See
  • acDeviceWaitOnEvent
  • acDeviceDeinitializeEvents

acDeviceDeinitializeEvents¶

AC_ERROR AC_API acDeviceDeinitializeEvents(acDevice hDevice)

DeinitializeEvents stops the underlying events engine from listening for messages, shutting it down and cleaning it up. It should be called only after the events infrastructure has been initialized (acDeviceInitializeEvents) and after all events have been processed (acDeviceWaitOnEvent).

Return
  • Type: AC_ERROR
  • Error code for the function
  • Returns AC_ERR_SUCCESS (0) on success
Parameters
  • hDevice:
    • Type: acDevice
    • [In] parameter
    • A device

See
  • acDeviceInitializeEvents
  • acDeviceWaitOnEvent

acDeviceWaitOnEvent¶

AC_ERROR AC_API acDeviceWaitOnEvent(acDevice hDevice, int64_t timeout)

WaitOnEvent waits for an event to occur in order to process its data. It must be called after the event infrastructure has been initialized (acDeviceInitializeEvents) and before it is deinitialized (acDeviceDeinitializeEvents). This will also trigger callbacks registered to event nodes.

Return
  • Type: AC_ERROR
  • Error code for the function
  • Returns AC_ERR_SUCCESS (0) on success
Parameters
  • hDevice:
    • Type: acDevice
    • [In] parameter
    • A device
  • timeout:
    • Type: uint64_t
    • [In] parameter
    • Maximum time to wait for an event

See
  • acDeviceInitializeEvents
  • acDeviceDeinitializeEvents

acDeviceRegisterImageCallback¶

AC_ERROR AC_API acDeviceRegisterImageCallback(acDevice hDevice, acCallback * phCallback, acImageCallbackFunction callbackFn, void * pParam)

acDeviceRegisterImageCallback registers the specified image callback function for the device. The optional user-specified data will be passed to the image callback function.

Return
  • Type: AC_ERROR
  • Error code for the function
  • Returns AC_ERR_SUCCESS (0) on success
Parameters
  • hDevice:
    • Type: acDevice
    • [In] parameter
    • A device
  • phCallback:
    • Type: acCallback*
    • [Out] parameter
    • Handle to registered acCallback
  • callbackFn:
    • Type: acImageCallbackFunction
    • [In] parameter
    • The image callback function to register
  • pParam:
    • Type: void*
    • [In] parameter
    • Pointer to optional user-specified callback data
